__Starting Structure__

1. create a VirtualEnvironment to isolate the project and activate it.

1. Pip install these dependencies for the project:
    ```

    $ pip install Django==1.9.3 elasticsearch requests django_extensions debug_toolbar django-debug-toolbar

    ```

1. Generate a new Django project: ```$ django-admin startproject project```

1. This is the default project structure generated by manage.py when starting a new project.
    ```

    .
    |_____project/
    |_____|_______init__.py
    |_____|_____settings.py
    |_____|_____urls.py
    |_____|_____wsgi.py
    |_____manage.py

    ```

Now that we have a project, the base files will need LOTS of tweaking.

1. Create an init file for the conf app: ```$ touch project/apps/conf/__init__.py```

1. Edit the ```conf/__init__.py``` file to match this:
    ```

    from .base import *

    try:
        from .local import *
    except ImportError:
        pass

    ```

1. Create the project/apps/core/models.py file: ```$ touch project/apps/core/models.py```

1. Populate the new models.py file as follows:
    ```

    from django.db import models
    from django.core.validators import MinValueValidator, MaxValueValidator

    class University(models.Model):
        name = models.CharField(max_length=255, unique=True)

    class Course(models.Model):
        name = models.CharField(max_length=255, unique=True)

    class Student(models.Model):
        YEAR_IN_SCHOOL_CHOICES = (
            ('FR', 'Freshman'),
            ('SO', 'Sophomore'),
            ('JR', 'Junior'),
            ('SR', 'Senior'),
        )
        # note: incorrect choice in MyModel.create leads to creation of incorrect record
        year_in_school = models.CharField(
            max_length=2, choices=YEAR_IN_SCHOOL_CHOICES)
        age = models.SmallIntegerField(
            validators=[MinValueValidator(1), MaxValueValidator(100)]
        )
        first_name = models.CharField(max_length=50)
        last_name = models.CharField(max_length=50)
        # various relationships models
        university = models.ForeignKey(University, null=True, blank=True)
        courses = models.ManyToManyField(Course, null=True, blank=True)

    ```

1. Make migrations for the new models: ```$ python project/manage.py makemigrations core```

1. This results in a new file: ```project/apps/core/migrations/0001_initial.py```

1. Make a directory for the db: ```$ mkdir project/db```

1. Build the db tables from the migration: ```$ python project/manage.py migrate```

1. Create a new file for the admin interface: ```$ tpuch project/apps/core/admin.py```

1. Edit the new admin.py file to match this:
    ```

    from django.contrib import admin
    from .models import University, Course, Student

    admin.site.register(University)
    admin.site.register(Course)
    admin.site.register(Student)

    ```

1. Create a superuser for the admin interface: ```$ python project/manage.py createsuperuser```
  - Provide a username and password as prompted.

1. Test the project: ```$ python project/manage.py runserver```
  - You can see the running project in a browser at: ```http://localhost:8000/admin/```
  - If you access ```http://localhost:8000``` you will get a 404 error because we have not define any urls to handle that route.

1. Login to the admin panel to verify everything is working.
